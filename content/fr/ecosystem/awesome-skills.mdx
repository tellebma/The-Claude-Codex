## TL;DR

- Les **skills** sont la couche de réutilisation la plus puissante de Claude Code : un dossier `.md` qui injecte un savoir-faire métier dans toutes tes sessions.
- 3 familles dominent : **packs complets** (`everything-claude-code`, `mattpocock/skills`), **skills mono-usage** (`caveman` pour les tokens, `graphify` pour la mémoire, `career-ops` pour le job hunting), et **skills design pro** (Impeccable, UI UX Pro Max, etc.).
- Les awesome lists `ComposioHQ/awesome-claude-skills` (59k★) et `sickn33/antigravity-awesome-skills` (37k★) servent de référence catalogue, à bookmarker.
- Conseil : installer **1 pack** + **2-3 skills mono-usage** ciblés sur tes vrais besoins. Pas 50 d'un coup.

<Callout type="info" title="Skills ≠ Plugins ≠ Agents">
Un **skill** est un fichier markdown qui ajoute une compétence à Claude Code (un savoir-faire, un workflow, des conventions). Un **plugin** est une intégration packagée distribuée via marketplace. Un **agent** est un sous-LLM délégué pour une tâche précise. Voir [`/skills/comparison`](/skills/comparison) pour le détail.
</Callout>

## Comment choisir

Trois étapes :

1. **Si tu débutes** : installer **un pack complet** (`mattpocock/skills` si tu fais du TypeScript, sinon `everything-claude-code` pour le couteau suisse). Travailler avec pendant 2-3 semaines avant d'ajouter quoi que ce soit.
2. **Si tu as une douleur précise** : aller chercher le skill mono-usage correspondant (`caveman` pour la facture qui explose, `claude-mem` pour le contexte perdu, `graphify` pour naviguer un gros codebase).
3. **Si tu veux comprendre l'écosystème** : passer 30 minutes sur `ComposioHQ/awesome-claude-skills` et `hesreallyhim/awesome-claude-code`. Tu ressortiras avec une vision claire.

<Callout type="warning" title="Ne pas empiler 12 packs">
Plusieurs packs en même temps = conflits de conventions + temps de chargement allongé + Claude qui ne sait plus quel skill prioriser. **Un pack à la fois**. Si tu veux essayer un nouveau pack, désactive le précédent.
</Callout>
