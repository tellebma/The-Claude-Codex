## Vue d'ensemble

Claude Code lit plusieurs variables d'environnement au démarrage. Elles permettent de configurer l'outil sans modifier les fichiers `settings.json`, ce qui est idéal pour les environnements CI/CD et les déploiements automatisés.

<Callout type="info" title="Priorité des variables d'environnement">
Les variables d'environnement ont généralement une priorité **égale ou inférieure** aux options explicitement définies dans `settings.json`. Consultez la documentation de chaque option pour connaître sa précédence exacte.
</Callout>

---

## Variables essentielles

### `ANTHROPIC_API_KEY`

| Champ | Valeur |
|-------|--------|
| **Description** | Clé API Anthropic pour l'authentification |
| **Requis** | Oui (sauf si connecté via `claude login`) |
| **Défaut** | Aucun |
| **Format** | `sk-ant-api03-...` |

```bash
# Dans votre ~/.bashrc, ~/.zshrc ou ~/.profile
export ANTHROPIC_API_KEY="sk-ant-api03-xxxxxxxxxxxx"

# Pour une session unique
ANTHROPIC_API_KEY="sk-ant-..." claude --print "bonjour"

# En CI/CD (GitHub Actions)
# Définir dans Settings > Secrets > ANTHROPIC_API_KEY
# Puis dans le workflow :
env:
  ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

<Callout type="warning" title="Sécurité de la clé API">
Ne committez jamais `ANTHROPIC_API_KEY` dans un fichier versionné. Utilisez un gestionnaire de secrets (AWS Secrets Manager, HashiCorp Vault, GitHub Secrets) en production. Faites une rotation de la clé immédiatement si elle est exposée.
</Callout>

---

### `CLAUDE_MODEL`

| Champ | Valeur |
|-------|--------|
| **Description** | Modèle Claude à utiliser par défaut |
| **Requis** | Non |
| **Défaut** | `claude-sonnet-4-6` |
| **Format** | Identifiant de modèle Anthropic |

```bash
# Utiliser Haiku pour les sessions rapides (3x moins cher)
export CLAUDE_MODEL="claude-haiku-4-5"

# Utiliser Opus pour les tâches complexes
export CLAUDE_MODEL="claude-opus-4-6"

# Surcharger pour une commande unique
CLAUDE_MODEL="claude-opus-4-6" claude --print "tâche complexe"
```

Identifiants de modèles courants :
- `claude-haiku-4-5` : Rapide et économique
- `claude-sonnet-4-6` : Équilibre recommandé
- `claude-opus-4-6` : Raisonnement le plus profond

---

### `MAX_THINKING_TOKENS`

| Champ | Valeur |
|-------|--------|
| **Description** | Budget de tokens alloués au raisonnement interne (Extended Thinking) |
| **Requis** | Non |
| **Défaut** | `31999` (quand l'Extended Thinking est activé) |
| **Format** | Nombre entier |

```bash
# Réduire le budget pour limiter les coûts
export MAX_THINKING_TOKENS=10000

# Budget maximum pour les tâches complexes
export MAX_THINKING_TOKENS=31999

# Désactiver le thinking pour une session (valeur minimale)
export MAX_THINKING_TOKENS=1000
```

Réduire cette valeur diminue la profondeur du raisonnement et les coûts associés. Pertinent uniquement quand `alwaysThinkingEnabled: true` ou quand Extended Thinking est activé manuellement.

## Variables réseau et proxy

### `ANTHROPIC_BASE_URL`

| Champ | Valeur |
|-------|--------|
| **Description** | URL de base personnalisée pour les appels API Anthropic |
| **Requis** | Non |
| **Défaut** | `https://api.anthropic.com` |
| **Format** | URL complète |

```bash
# Proxy d'entreprise
export ANTHROPIC_BASE_URL="https://proxy.mon-entreprise.com/anthropic"

# Serveur local (développement/test)
export ANTHROPIC_BASE_URL="http://localhost:8080"

# Compatible avec les variables du settings.json
# (ANTHROPIC_BASE_URL surcharge customApiUrl)
```

---

### `HTTP_PROXY` / `HTTPS_PROXY`

| Champ | Valeur |
|-------|--------|
| **Description** | Proxy HTTP/HTTPS pour toutes les connexions sortantes |
| **Requis** | Non |
| **Défaut** | Aucun |
| **Format** | `http://[user:pass@]host:port` |

```bash
# Proxy sans authentification
export HTTP_PROXY="http://proxy.entreprise.com:8080"
export HTTPS_PROXY="http://proxy.entreprise.com:8080"

# Proxy avec authentification
export HTTPS_PROXY="http://utilisateur:motdepasse@proxy.entreprise.com:8080"

# Proxy SOCKS5
export HTTPS_PROXY="socks5://proxy.entreprise.com:1080"
```

---

### `NO_PROXY`

| Champ | Valeur |
|-------|--------|
| **Description** | Domaines exclus du proxy (liste séparée par virgules) |
| **Requis** | Non |
| **Défaut** | Aucun |
| **Format** | Liste de domaines séparés par virgules |

```bash
# Exclure le réseau local du proxy
export NO_PROXY="localhost,127.0.0.1,*.intranet.entreprise.com"

# Combiner avec HTTPS_PROXY
export HTTPS_PROXY="http://proxy.entreprise.com:8080"
export NO_PROXY="localhost,127.0.0.1"
```

## Variables de comportement

### `DISABLE_AUTOUPDATER`

| Champ | Valeur |
|-------|--------|
| **Description** | Désactive la vérification et l'installation automatique des mises à jour |
| **Requis** | Non |
| **Défaut** | Non défini (mise à jour auto activée) |
| **Format** | `1` ou `true` pour désactiver |

```bash
# Désactiver les mises à jour automatiques
export DISABLE_AUTOUPDATER=1

# Utile en CI/CD pour figer la version de Claude Code
# et éviter les mises à jour non souhaitées en production
```

Recommandé en CI/CD pour la reproductibilité des builds. Utilisez une version fixe de Claude Code dans vos pipelines.

---

### `CLAUDE_CODE_MAX_OUTPUT_TOKENS`

| Champ | Valeur |
|-------|--------|
| **Description** | Nombre maximum de tokens dans la réponse générée |
| **Requis** | Non |
| **Défaut** | Limite du modèle (variable selon le modèle) |
| **Format** | Nombre entier |

```bash
# Limiter les réponses longues
export CLAUDE_CODE_MAX_OUTPUT_TOKENS=4096

# Maximum pour les tâches nécessitant de longues réponses
export CLAUDE_CODE_MAX_OUTPUT_TOKENS=16000
```

---

### `CLAUDE_CONFIG_DIR`

| Champ | Valeur |
|-------|--------|
| **Description** | Répertoire alternatif pour les fichiers de configuration Claude Code |
| **Requis** | Non |
| **Défaut** | `~/.claude/` |
| **Format** | Chemin absolu vers un répertoire |

```bash
# Utiliser un répertoire de configuration alternatif
export CLAUDE_CONFIG_DIR="/opt/claude-config"

# Utile pour les environnements multi-utilisateurs
# ou les configurations isolées par projet
export CLAUDE_CONFIG_DIR="$(pwd)/.claude-config"

# En CI/CD, pour une configuration complètement isolée
export CLAUDE_CONFIG_DIR="/tmp/claude-ci-config"
```

## Exemples de configuration par contexte

### Développement local

```bash
# ~/.zshrc ou ~/.bashrc
export ANTHROPIC_API_KEY="sk-ant-api03-..."
export CLAUDE_MODEL="claude-sonnet-4-6"

# Optionnel : Extended Thinking avec budget réduit
export MAX_THINKING_TOKENS=10000
```

### CI/CD (GitHub Actions)

```yaml
# .github/workflows/claude.yml
jobs:
  claude-task:
    runs-on: ubuntu-latest
    env:
      ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
      CLAUDE_MODEL: claude-haiku-4-5
      DISABLE_AUTOUPDATER: "1"
      CLAUDE_CODE_MAX_OUTPUT_TOKENS: "4096"
    steps:
      - uses: actions/checkout@v4
      - run: npm install -g @anthropic-ai/claude-code
      - run: claude --print --max-turns 5 "Vérifie la qualité du code"
```

### Environnement d'entreprise avec proxy

```bash
# /etc/environment ou profil système
ANTHROPIC_API_KEY="sk-ant-..."
ANTHROPIC_BASE_URL="https://proxy-ia.mon-entreprise.com"
HTTPS_PROXY="http://proxy-web.mon-entreprise.com:8080"
NO_PROXY="localhost,127.0.0.1,*.interne.mon-entreprise.com"
DISABLE_AUTOUPDATER=1
CLAUDE_CONFIG_DIR="/opt/claude-code/config"
```

## Référence étendue : variables moins courantes

### `ANTHROPIC_AUTH_TOKEN`

Token d'authentification alternatif utilisé par certains proxys d'entreprise qui n'acceptent pas l'en-tête standard `ANTHROPIC_API_KEY`. Quand il est défini, Claude Code l'envoie dans `Authorization: Bearer <token>`.

```bash
export ANTHROPIC_AUTH_TOKEN="bearer-token-du-proxy"
```

### `ANTHROPIC_MODEL`

Alias accepté en plus de `CLAUDE_MODEL`. Certains tutoriels et outils tiers utilisent `ANTHROPIC_MODEL` ; les deux fonctionnent et pointent vers le même réglage.

```bash
export ANTHROPIC_MODEL="claude-sonnet-4-5"
```

### `CLAUDE_CODE_USE_BEDROCK` / `CLAUDE_CODE_USE_VERTEX`

Route les appels API via AWS Bedrock ou Google Vertex AI plutôt que directement vers l'API Anthropic. Nécessaire quand votre organisation impose l'un de ces fournisseurs.

```bash
# AWS Bedrock
export CLAUDE_CODE_USE_BEDROCK=1
export AWS_REGION="us-east-1"

# Google Vertex AI
export CLAUDE_CODE_USE_VERTEX=1
export CLOUD_ML_REGION="us-central1"
export ANTHROPIC_VERTEX_PROJECT_ID="votre-projet-gcp"
```

### `BASH_DEFAULT_TIMEOUT_MS` / `BASH_MAX_TIMEOUT_MS`

Contrôle combien de temps l'outil Bash attend avant de tuer une commande longue. Utile pour les jobs CI qui exécutent des builds lents.

```bash
export BASH_DEFAULT_TIMEOUT_MS=120000   # 2 min par défaut
export BASH_MAX_TIMEOUT_MS=600000       # 10 min maximum
```

### `BASH_MAX_OUTPUT_LENGTH`

Sortie maximale (en caractères) capturée pour une commande Bash. Empêche les logs énormes de saturer le contexte.

```bash
export BASH_MAX_OUTPUT_LENGTH=30000
```

### `MCP_TIMEOUT` / `MCP_TOOL_TIMEOUT`

Timeouts (ms) pour la connexion au serveur MCP et pour chaque appel d'outil. Augmentez ces valeurs pour les MCPs distants lents.

```bash
export MCP_TIMEOUT=30000         # handshake de démarrage
export MCP_TOOL_TIMEOUT=60000    # appel d'outil individuel
```

### `MAX_MCP_OUTPUT_TOKENS`

Budget de tokens pour la sortie combinée de tous les appels d'outils MCP dans un même tour.

```bash
export MAX_MCP_OUTPUT_TOKENS=25000
```

### `CLAUDE_CODE_DISABLE_TERMINAL_TITLE`

Empêche Claude Code de mettre à jour le titre de la fenêtre du terminal. Utile pour les utilisateurs tmux et les prompts personnalisés.

```bash
export CLAUDE_CODE_DISABLE_TERMINAL_TITLE=1
```

### `DISABLE_TELEMETRY` / `DISABLE_ERROR_REPORTING`

Désactive la télémétrie anonyme et les rapports d'erreur Sentry respectivement. Les deux sont opt-out.

```bash
export DISABLE_TELEMETRY=1
export DISABLE_ERROR_REPORTING=1
```

## Variables d'environnement dans VS Code (`claudecode.environmentVariables`)

L'extension officielle Claude Code pour VS Code expose un réglage `claudecode.environmentVariables` qui injecte les variables dans chaque processus Claude Code lancé depuis l'éditeur. C'est la méthode la plus propre pour définir des valeurs par projet sans polluer votre shell global.

Ouvrez `settings.json` (User ou Workspace) et ajoutez :

```json
{
  "claudecode.environmentVariables": {
    "ANTHROPIC_API_KEY": "sk-ant-api03-...",
    "ANTHROPIC_BASE_URL": "https://proxy.entreprise.com/anthropic",
    "CLAUDE_MODEL": "claude-sonnet-4-5",
    "DISABLE_AUTOUPDATER": "1",
    "NO_PROXY": "localhost,127.0.0.1,*.intranet"
  }
}
```

Un fichier `.vscode/settings.json` au niveau Workspace est idéal pour committer la config non-secrète (modèle, proxy, timeouts) tout en gardant `ANTHROPIC_API_KEY` dans les User settings uniquement.

<Callout type="warning" title="Ne committez jamais de secrets">
Le `settings.json` Workspace est généralement committé sur git. Gardez `ANTHROPIC_API_KEY` dans les User settings ou utilisez `${env:ANTHROPIC_API_KEY}` pour lire depuis l'environnement shell plutôt que de le hardcoder.
</Callout>

## Ordre de priorité (quelle source gagne ?)

Quand une même variable est définie à plusieurs endroits, Claude Code les résout dans cet ordre — le dernier gagne :

1. Environnement système (`~/.bashrc`, `~/.zshrc`, `/etc/environment`)
2. Export shell par session
3. Fichier `.env` dans le dossier du projet
4. `claudecode.environmentVariables` de VS Code (User puis Workspace)
5. Option `settings.json` équivalente (ex. `customApiUrl` vs `ANTHROPIC_BASE_URL`)
6. Flag CLI (`--model`, `--max-turns`, etc.)

Les flags CLI gagnent toujours. Les valeurs `settings.json` surchargent en général les variables d'environnement quand un équivalent direct existe — vérifiez la documentation de chaque option pour être certain.

## FAQ

### Où définir `ANTHROPIC_API_KEY` ?

Pour le dev local, exportez-la dans `~/.zshrc` ou `~/.bashrc`. Pour le CI/CD (GitHub Actions, GitLab CI), stockez-la comme secret chiffré et passez-la via le bloc `env:`. Ne la committez jamais sur git et ne la hardcodez pas dans `settings.json`.

### Pourquoi mon `ANTHROPIC_BASE_URL` est ignoré ?

Vérifiez que `HTTPS_PROXY` ne surcharge pas la route. Si vous utilisez un proxy d'entreprise, définissez à la fois `ANTHROPIC_BASE_URL` (vers l'endpoint compatible Anthropic) et `NO_PROXY` (en incluant cet endpoint) pour que le proxy n'avale pas la requête.

### Quelle différence entre `CLAUDE_MODEL` et `ANTHROPIC_MODEL` ?

Ce sont des alias — Claude Code accepte les deux. Utilisez `CLAUDE_MODEL` dans les nouvelles configurations pour rester cohérent avec la doc officielle.

### Comment désactiver Extended Thinking via variable d'environnement ?

Définissez `MAX_THINKING_TOKENS=1000` (le minimum). Cela désactive de fait le raisonnement profond tout en gardant la fonctionnalité opérationnelle. Pour le désactiver complètement, retirez `alwaysThinkingEnabled: true` de `settings.json`.

### Comment figer une version spécifique de Claude Code en CI ?

Définissez `DISABLE_AUTOUPDATER=1` et installez une version précise : `npm install -g @anthropic-ai/claude-code@x.y.z`. Cela garantit la reproductibilité des builds et évite les surprises d'auto-update.

### Les variables d'environnement fuitent-elles dans le contexte de Claude ?

Non. Les variables d'environnement configurent le client Claude Code lui-même, pas la conversation. Claude ne voit jamais `ANTHROPIC_API_KEY` ou les réglages proxy sauf si vous les incluez explicitement dans un prompt.
