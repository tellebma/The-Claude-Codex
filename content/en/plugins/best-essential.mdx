## Les ressources que tout développeur Claude Code devrait connaître

Ces ressources communautaires sont les **fondamentaux** de l'écosystème Claude Code. Elles enrichissent considérablement votre expérience quotidienne : agents spécialisés, documentation à jour, analyse de codebase et amélioration de prompts.

<Callout type="info" title="Extensions communautaires, pas des plugins officiels">
Les ressources présentées sur cette page sont des **projets communautaires**, non officiellement publiés par Anthropic. Elles s'installent manuellement (copie de fichiers, npm, ou `/install-github`). Il n'existe pas de commande `/plugin install` dans Claude Code.
</Callout>

---

## Everything Claude Code : La collection complète

**Projet communautaire** | Catégorie : Agents, Rules & Skills | Auteur : [punkpeye](https://github.com/punkpeye)

[Everything Claude Code](https://github.com/punkpeye/everything-claude-code) est l'une des collections communautaires les plus connues de l'écosystème. Elle regroupe une **collection d'agents spécialisés, de règles et de skills** prêts à l'emploi qui couvrent les cas d'usage les plus courants du développement.

<Callout type="info" title="Projet communautaire open source">
Everything Claude Code est hébergé sur GitHub à l'adresse [github.com/punkpeye/everything-claude-code](https://github.com/punkpeye/everything-claude-code). C'est un projet de la communauté, non affilié à Anthropic.
</Callout>

### Ce qu'il apporte

- **Agents spécialisés** : `senior-dev`, `architect`, `reviewer`, `debugger`, `optimizer`, chacun avec une expertise et un style de réponse adaptés
- **Rules de workflow** : conventions de code, bonnes pratiques, guidelines de revue de PR
- **Skills** : commandes slash pour les tâches courantes (TDD, refactoring, documentation)
- **Templates CLAUDE.md** : fichiers de configuration optimisés pour différents stacks

### Installation

```bash
# Via /install-github (recommandé)
/install-github punkpeye/everything-claude-code

# Ou manuellement
git clone https://github.com/punkpeye/everything-claude-code /tmp/ecc
mkdir -p .claude/agents .claude/rules .claude/commands
cp -r /tmp/ecc/agents/ .claude/
cp -r /tmp/ecc/rules/ .claude/
cp -r /tmp/ecc/commands/ .claude/
```

### Use cases

- **Revue de code complète** : *"En tant qu'agent reviewer, analyse ma dernière PR et suggère des améliorations"*
- **Architecture d'un nouveau module** : *"En tant qu'architect, aide-moi à concevoir le système de notification"*
- **Debugging avancé** : *"En tant que debugger, aide-moi à comprendre pourquoi les tests E2E échouent sur CI"*
- **Optimisation** : *"En tant qu'optimizer, analyse les performances de mon API et propose des améliorations"*

<Callout type="tip" title="Adapter les agents à votre projet">
Les agents d'Everything Claude Code sont des fichiers Markdown : vous pouvez les modifier librement pour les adapter aux conventions de votre équipe. Forkez le repo et personnalisez selon vos besoins.
</Callout>

## Context7 : La documentation toujours à jour

**MCP Server** | Catégorie : Documentation | Auteur : [Upstash](https://upstash.com/)

Context7 résout un problème majeur : les **connaissances obsolètes**. Claude Code a une date de coupure pour ses connaissances. Context7 est un **MCP server** (pas un plugin) qui lui permet d'accéder à la **documentation la plus récente** de n'importe quelle bibliothèque ou framework, en temps réel.

<Callout type="info" title="Context7 est un MCP, pas un plugin">
Context7 s'installe comme un **serveur MCP** dans votre `settings.json`, pas comme un fichier Markdown. Son package npm est `@upstash/context7-mcp`. Il est développé par Upstash et est open source sur [github.com/upstash/context7](https://github.com/upstash/context7).
</Callout>

### Installation

```bash
# Via la commande Claude
claude mcp add context7 -- npx -y @upstash/context7-mcp
```

### Configuration JSON

```json
{
  "mcpServers": {
    "context7": {
      "command": "npx",
      "args": ["-y", "@upstash/context7-mcp"]
    }
  }
}
```

### Use cases

- **API inconnue** : *"Utilise Context7 pour trouver comment utiliser useOptimistic de React 19"*
- **Migration de version** : *"Quels breaking changes dans Next.js 15 par rapport à Next.js 14 ?"*
- **Vérification de syntaxe** : *"Vérifie dans la doc Tailwind si la classe text-balance est disponible en v3"*

## Repomix : Analyse complète de codebase

**Outil CLI npm** | Catégorie : Analyse | Auteur : [yamadashy](https://github.com/yamadashy)

[Repomix](https://github.com/yamadashy/repomix) est un **outil CLI** (pas un plugin) qui condense une codebase entière en un seul fichier structuré, optimisé pour être passé à Claude Code comme contexte. Au lieu de lire fichier par fichier, Claude Code obtient une vue d'ensemble complète et organisée de votre projet.

<Callout type="info" title="Repomix sur npm">
Repomix est disponible sur [npmjs.com/package/repomix](https://www.npmjs.com/package/repomix). Son repo source est sur [github.com/yamadashy/repomix](https://github.com/yamadashy/repomix).
</Callout>

### Ce qu'il apporte

- **Condensation intelligente** : génère un fichier texte unique contenant tout votre code, dans un format optimisé pour les LLMs
- **Filtrage configurable** : exclut `node_modules`, fichiers binaires, et tout ce que vous configurez
- **Statistiques** : compte les tokens pour estimer l'utilisation du contexte
- **Multiple formats** : sortie en texte brut, XML ou Markdown

### Installation et utilisation

```bash
# Utilisation sans installation (npx)
npx repomix

# Ou installation globale
npm install -g repomix

# Analyser le répertoire courant
repomix

# Analyser en excluant certains fichiers
repomix --ignore "node_modules/**,*.lock,dist/**"

# Sortie dans un fichier spécifique
repomix --output context.txt

# Passer le résultat à Claude Code :
# "Voici ma codebase [colle le contenu de context.txt]. Explique l'architecture."
```

### Use cases

- **Onboarding** : *"Analyse cette codebase et donne-moi un résumé de l'architecture, des technologies et des conventions"*
- **Impact analysis** : *"Si je modifie ce module, quels autres fichiers seront probablement impactés ?"*
- **Détection de dette technique** : *"Identifie les zones de dette technique dans ce projet et priorise les refactorings"*
- **Documentation** : *"Génère une documentation d'architecture basée sur l'analyse du code"*

## Prompt Improver : Des prompts optimisés

**Projet Anthropic** | Catégorie : Prompting | Auteur : [Anthropic](https://github.com/anthropics)

[Prompt Improver](https://github.com/anthropics/prompt-improver) est un outil développé par Anthropic qui aide à rédiger de meilleurs prompts. Ce n'est pas un plugin au sens d'une extension installée dans Claude Code, c'est un outil web ou CLI qui analyse vos prompts et propose des améliorations.

<Callout type="warning" title="Vérifiez l'existence du dépôt">
Prompt Improver est un outil d'Anthropic. Le dépôt GitHub exact peut avoir changé de nom ou de structure. Vérifiez sur [github.com/anthropics](https://github.com/anthropics) avant de cloner. Anthropic propose également un outil d'amélioration de prompts intégré à la [Console Anthropic](https://console.anthropic.com).
</Callout>

### Ce qu'il apporte

- **Analyse de structure** : identifie si votre prompt manque de contexte, de contraintes ou de format cible
- **Suggestions concrètes** : reformule les parties vagues en instructions précises et structurées
- **Best practices** : applique les techniques de prompting recommandées par Anthropic (XML tags, chain-of-thought, exemples)

### Use cases manuel

Même sans outil automatisé, vous pouvez améliorer vos prompts dans Claude Code en appliquant ces principes directement :

- **Prompt vague** : *"Fais un formulaire de contact"*
- **Prompt amélioré** : *"Crée un formulaire de contact React avec les champs nom, email et message. Validation Zod côté client. Gestion d'erreurs explicite. État de chargement pendant la soumission. Message de succès après envoi. Composants accessibles (WCAG AA) avec attributs aria appropriés. Tests unitaires avec React Testing Library."*
