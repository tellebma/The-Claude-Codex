The Claude Codex a vu le jour debut 2026. En quelques semaines, le site est passe d'une landing page a un guide complet de plus de 50 pages couvrant l'installation, les MCP, les Skills, les agents, le prompting, l'entreprise et bien plus. Voici un bilan de ce qui a ete fait, ce qui arrive, et comment vous pouvez contribuer.

## Ce qui a ete realise

### Phase 1 : Fondations (janvier-fevrier 2026)

Les premieres semaines ont pose les bases techniques et editoriales du projet.

**Infrastructure :**
- Stack technique : Next.js 14 (App Router), TypeScript strict, Tailwind CSS, MDX
- Export statique (SSG) pour des performances maximales
- Deploiement Docker (Nginx Alpine), image < 50 MB
- SEO complet : meta-donnees, JSON-LD, sitemap, canonical URLs
- Dark mode / Light mode, design mobile-first, score Lighthouse > 90

**Contenu initial :**
- Landing page avec presentation des fonctionnalites principales
- Section "Premiers pas" : installation, configuration, premier projet
- Section MCP : comprehension, installation, top MCP par categorie
- Section Prompting : bases, directives, templates, erreurs a eviter
- Section Skills : concepts, top skills, creation de skill custom
- Section Agents : concepts, creation, agent teams, orchestration
- Vision & Futur : page d'overview avec tendances et roadmap

### Phase 2 : Approfondissement (mars 2026)

Le contenu s'est enrichi pour couvrir les besoins des debutants comme des experts.

**Nouveaux contenus :**
- Parcours debutants : pre-requis zero, glossaire 40+ termes, FAQ
- Articles editoriaux : couts reels, mythes, bonnes pratiques securite
- Section Entreprise : securite, adoption, TCO, FAQ, gouvernance
- Section Avance : hooks, headless/CI-CD, multi-provider
- Reference technique : cheatsheet, CLI, settings.json, variables d'environnement
- Section Limites : limitations connues, comparatifs Copilot et Cursor, quand ne pas utiliser
- Parcours personas : developpeur, lead, non-dev, freelance, etudiant
- MCP avance : creation TypeScript, creation Python, protocole avance
- Configurateur interactif : generateur de configuration en quelques clics
- Section Vision : pages thematiques sur l'IA et les metiers, tendances 2026

**Ameliorations techniques :**
- Recherche integree avec normalisation des accents
- Navigation inter-pages avec previous/next
- Table des matieres auto-generee sur chaque page
- Breadcrumb et donnees structurees sur toutes les sous-pages

## Ce qui arrive

Voici les fonctionnalites et contenus prevus pour les prochains mois. Les priorites peuvent evoluer en fonction des retours de la communaute.

### Q2 2026 : Interactivite et communaute

<Callout type="info" title="En cours de developpement">
Ces fonctionnalites sont en preparation. Les dates sont indicatives et peuvent changer.
</Callout>

**Tutoriels interactifs**

L'objectif est de proposer des exercices directement dans le navigateur. Vous pourrez ecrire un prompt, voir le resultat simule, et comprendre pourquoi une formulation fonctionne mieux qu'une autre, sans quitter le site.

**Guides video**

Des walkthroughs visuels pour les sujets les plus demandes : premiere installation, creation d'un MCP, configuration d'agents, workflows multi-outils. Format court (5-10 minutes), sur YouTube et integres dans les pages.

**Systeme de contributions communautaires**

Un processus clair pour soumettre des contenus : recettes, workflows, retours d'experience. Les contributions seront relues et integrees dans le guide avec credit a l'auteur.

### Q3 2026 : Repertoire et outils

**Catalogue MCP & Skills**

Un repertoire cherchable de tous les MCP et Skills disponibles, avec des fiches detaillees : description, auteur, date de derniere mise a jour, note de securite, instructions d'installation.

**Playground de prompting**

Un espace pour tester des prompts et comparer les resultats entre differentes formulations. Utile pour apprendre le prompting sans consommer de tokens.

### Q4 2026 : Internationalisation

**Support multilingue**

Le guide traduit en anglais (priorite 1), puis en espagnol, arabe et autres langues en fonction de la demande. L'objectif : rendre le contenu accessible au-dela de la francophonie.

## Comment contribuer

The Claude Codex est un projet ouvert. Toute contribution est la bienvenue, de la correction d'une faute de frappe a la redaction d'un chapitre complet.

### Reporter un probleme

Vous avez repere une erreur, un lien casse ou une information obsolete ?

<Steps>
<Step title="Ouvrez une issue" stepNumber={1}>
Allez sur le [repository GitHub](https://github.com/tellebma/The-Claude-Codex) et cliquez sur "New Issue".
</Step>
<Step title="Decrivez le probleme" stepNumber={2}>
Indiquez la page concernee, ce qui est incorrect et, si possible, ce que devrait etre la bonne information.
</Step>
<Step title="On s'en occupe" stepNumber={3}>
L'equipe trie les issues regulierement. Les corrections simples sont souvent publiees dans la journee.
</Step>
</Steps>

### Proposer du contenu

Vous avez un workflow, une recette ou un retour d'experience a partager ?

<Steps>
<Step title="Forkez le repository" stepNumber={1}>
Clonez le projet en local et creez une branche avec un nom descriptif (`feat/mon-article`).
</Step>
<Step title="Ecrivez votre contenu en MDX" stepNumber={2}>
Placez votre fichier dans `content/` avec un frontmatter complet (title, description, section, order). Consultez les fichiers existants pour le format.
</Step>
<Step title="Soumettez une pull request" stepNumber={3}>
Poussez votre branche et ouvrez une PR. Decrivez ce que vous ajoutez et pourquoi. L'equipe fera une revue et vous guidera si des ajustements sont necessaires.
</Step>
</Steps>

### Conventions de contribution

Quelques regles pour que les contributions s'integrent bien :

- **Langue :** francais, ton accessible et direct. Pas de jargon inutile, pas de formules creuses
- **Style :** pas de tiret cadratin, pas de formulations qui sonnent comme une IA ("Il est important de noter que...", "Plongez dans...")
- **Structure :** introduction claire, corps structure avec des titres h2/h3, conclusion avec "Prochaines etapes"
- **MDX :** utilisez les composants disponibles (`<Callout>`, `<Steps>`, `<Card>`, `<CodeBlock>`) quand c'est pertinent
- **Frontmatter :** `title` et `description` sont obligatoires. Ajoutez `order` pour le positionnement dans la navigation

<Callout type="tip" title="Besoin d'aide ?">
Si vous n'etes pas a l'aise avec Git ou MDX, ouvrez simplement une issue avec votre contenu en texte brut. On s'occupera de la mise en forme.
</Callout>

## Vision a long terme

L'ambition de The Claude Codex est de devenir la reference francophone pour Claude Code et, plus largement, pour l'utilisation de l'IA dans le developpement.

**Ce que ca veut dire concretement :**

- **Completude :** couvrir tous les sujets, du debutant absolu a l'expert qui deploie en entreprise. Pas de trou dans le parcours.
- **Qualite :** chaque page doit etre utile, a jour et bien ecrite. Mieux vaut 50 pages excellentes que 200 pages mediocres.
- **Accessibilite :** le site doit rester gratuit, sans paywall, sans tracking intrusif. Le savoir technique ne devrait pas etre un privilege.
- **Communaute :** a terme, le guide devrait etre autant alimente par la communaute que par l'equipe fondatrice. Les meilleures ressources techniques open-source fonctionnent sur ce modele.
- **Honnetete :** on parle des limites autant que des forces. Pas de marketing, pas de hype. Si un outil ne fonctionne pas bien pour un cas d'usage, on le dit.

### Ce que ce projet n'est pas

Pour etre transparent sur les limites volontaires du projet :

- Ce n'est pas un site commercial. Pas de produit a vendre, pas de cours payant derriere un formulaire
- Ce n'est pas de la documentation officielle Anthropic. C'est un guide independant, avec un point de vue editorial
- Ce n'est pas un site d'actualites IA. On ne couvre pas chaque release, on se concentre sur ce qui est utile durablement
