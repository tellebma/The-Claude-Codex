## Votre profil

Vous ecrivez du code au quotidien. Vous connaissez le terminal, Git, et au moins un langage de programmation. Vous cherchez un outil qui accelere votre workflow sans sacrifier la qualite du code.

Que vous soyez junior ou senior, ce parcours vous emmene de l'installation a la maitrise des fonctionnalites avancees de Claude Code.

<Callout type="info" title="Temps estime">
Comptez 2 a 3 heures pour parcourir l'ensemble des etapes. Vous pouvez aussi les repartir sur plusieurs jours.
</Callout>

## Votre itineraire

<Steps>
<Step title="Comprendre Claude Code" stepNumber={1}>

Avant d'installer quoi que ce soit, prenez 5 minutes pour comprendre ce que Claude Code fait (et ne fait pas). Cette page compare Claude Code a Copilot et Cursor, et explique son fonctionnement en ligne de commande.

**Lire** : [Qu'est-ce que Claude Code ?](/getting-started/what-is-claude-code)

</Step>

<Step title="Installer et configurer" stepNumber={2}>

L'installation prend moins de 5 minutes si vous avez deja Node.js. Sinon, le guide couvre tout depuis zero.

**Lire** : [Prerequis et installation](/getting-started/installation), puis [Configuration de l'environnement](/getting-started/environment-setup)

</Step>

<Step title="Creer votre premier projet" stepNumber={3}>

Un tutoriel concret pour prendre en main Claude Code sur un vrai projet. Vous verrez comment formuler vos demandes et iterer sur le resultat.

**Lire** : [Premier projet pas a pas](/getting-started/first-project)

</Step>

<Step title="Maitriser le prompting" stepNumber={4}>

Le prompting est la competence cle. Un bon prompt fait la difference entre un resultat moyen et un resultat excellent. Commencez par les bases, puis explorez les patterns avances.

**Lire** : [Les bases du prompting](/prompting/basics), puis [Directives avancees](/prompting/directives), puis [Prompting avance](/prompting/advanced)

</Step>

<Step title="Configurer CLAUDE.md" stepNumber={5}>

Le fichier CLAUDE.md est la memoire persistante de Claude Code. Bien configure, il transforme chaque session en continuation intelligente de la precedente.

**Lire** : [Guide CLAUDE.md](/prompting/claude-md)

</Step>

<Step title="Connecter vos outils avec les MCP" stepNumber={6}>

Les MCP connectent Claude Code a GitHub, vos bases de donnees, Playwright, Sentry et des dizaines d'autres outils. C'est la que Claude Code devient vraiment puissant.

**Lire** : [Comprendre les MCP](/mcp/what-are-mcps), puis [Top MCP developpement](/mcp/best-development), puis [Premier workflow MCP](/mcp/first-workflow)

</Step>

<Step title="Automatiser avec les agents" stepNumber={7}>

Les agents delegent des taches complexes a des sous-agents specialises : code review, TDD, securite, refactoring. C'est le niveau au-dessus du prompting classique.

**Lire** : [Comprendre les agents](/agents/what-are-agents), puis [Creer un subagent](/agents/create-subagent), puis [Top agents par cas d'usage](/agents/best-agents)

</Step>

<Step title="Explorer les fonctionnalites avancees" stepNumber={8}>

Hooks pour automatiser des actions, mode headless pour vos pipelines CI/CD, multi-provider pour choisir le bon modele selon la tache.

**Lire** : [Systeme de Hooks](/advanced/hooks), puis [Mode Headless et CI/CD](/advanced/headless-ci)

</Step>
</Steps>

## Prochaines etapes

Votre parcours developpeur est termine. Voici quelques pistes pour aller plus loin :

- **Optimisez vos couts** : consultez [Couts reels de Claude Code](/content/couts-reels-claude-code) pour comprendre votre consommation de tokens
- **Explorez les plugins** : la section [Plugins](/plugins) complete les MCP avec des outils specialises
- **Creez vos propres MCP** : si vous avez des besoins specifiques, suivez le tutoriel [Creer un MCP en TypeScript](/mcp/create-mcp-typescript) ou [en Python](/mcp/create-mcp-python)
- **Gardez la reference sous la main** : le [Cheatsheet](/reference/cheatsheet) resume toutes les commandes en une page
