## Votre profil

Vous apprenez a coder, que ce soit en formation, en ecole d'ingenieur, en bootcamp ou en autodidacte. Claude Code peut devenir votre meilleur outil d'apprentissage, a condition de l'utiliser correctement.

Ce parcours vous guide step by step : d'abord les fondamentaux, puis des projets concrets, et enfin les pratiques professionnelles qui feront la difference sur votre CV.

<Callout type="warning" title="Utiliser Claude Code pour apprendre, pas pour tricher">
Claude Code peut ecrire du code a votre place. C'est tentant, mais ca ne vous apprendra rien. La bonne approche : demandez-lui d'expliquer, de corriger vos erreurs et de vous guider. Utilisez-le comme un tuteur, pas comme un copier-coller.
</Callout>

## Votre itineraire

<Steps>
<Step title="Comprendre les bases" stepNumber={1}>

Si le terminal, Node.js ou les cles API sont des concepts flous, commencez ici. Cette page explique tout sans presupposer de connaissances prealables.

**Lire** : [Pre-requis pour grands debutants](/getting-started/prerequisites-zero)

</Step>

<Step title="Installer Claude Code" stepNumber={2}>

L'installation prend quelques minutes. Suivez le guide step by step : meme si c'est votre premiere fois avec npm, vous y arriverez.

**Lire** : [Prerequis et installation](/getting-started/installation), puis [Configuration de l'environnement](/getting-started/environment-setup)

</Step>

<Step title="Realiser votre premier projet" stepNumber={3}>

Rien ne vaut la pratique. Ce tutoriel vous fait creer un vrai site web en quelques minutes. Observez comment Claude Code travaille : c'est deja une forme d'apprentissage.

**Lire** : [Premier projet step by step](/getting-started/first-project)

</Step>

<Step title="Apprendre a bien poser vos questions" stepNumber={4}>

Le prompting est une competence recherchee en entreprise. Apprenez a formuler des demandes claires et precises, c'est utile bien au-dela de Claude Code.

**Lire** : [Les bases du prompting](/prompting/basics), puis [Erreurs a eviter](/prompting/mistakes)

</Step>

<Step title="Comprendre CLAUDE.md" stepNumber={5}>

Le fichier CLAUDE.md definit le comportement de Claude Code pour chaque projet. Apprenez a le configurer : c'est une competence directement transferable en entreprise.

**Lire** : [Guide CLAUDE.md](/prompting/claude-md)

</Step>

<Step title="Decouvrir les MCP" stepNumber={6}>

Les MCP sont le mecanisme central d'extension de Claude Code. Comprendre leur fonctionnement vous donne une longueur d'avance sur les concepts d'API et d'integration.

**Lire** : [Comprendre les MCP](/mcp/what-are-mcps), puis [Installer et configurer](/mcp/setup)

</Step>

<Step title="Explorer les agents et le TDD" stepNumber={7}>

Les agents et le Test-Driven Development sont des pratiques professionnelles. Claude Code les rend accessibles, meme pour un etudiant. Apprenez a les utiliser tot, vous serez en avance sur le marche.

**Lire** : [Comprendre les agents](/agents/what-are-agents), puis [Top agents par cas d'usage](/agents/best-agents)

</Step>

<Step title="Comparer Skills, MCP et Plugins" stepNumber={8}>

Pour vos entretiens et vos projets, savoir distinguer ces trois mecanismes d'extension montre que vous comprenez l'architecture de l'outil en profondeur.

**Lire** : [Skills vs MCP vs Plugins](/skills/comparison)

</Step>
</Steps>
