## TL;DR

- **Skills** are Claude Code's most powerful reuse layer: a `.md` folder that injects domain expertise into every session.
- 3 families dominate: **complete packs** (`everything-claude-code`, `mattpocock/skills`), **single-purpose skills** (`caveman` for tokens, `graphify` for memory, `career-ops` for job hunting), and **pro design skills** (Impeccable, UI UX Pro Max, etc.).
- The awesome lists `ComposioHQ/awesome-claude-skills` (59k★) and `sickn33/antigravity-awesome-skills` (37k★) serve as reference catalogs — bookmark them.
- Advice: install **1 pack** + **2-3 single-purpose skills** targeted at your actual pain points. Not 50 at once.

<Callout type="info" title="Skills ≠ Plugins ≠ Agents">
A **skill** is a markdown file that adds a competence to Claude Code (a know-how, workflow, or convention). A **plugin** is a packaged integration distributed via marketplace. An **agent** is a delegated sub-LLM for a specific task. See [`/skills/comparison`](/skills/comparison) for the detail.
</Callout>

## How to choose

Three steps:

1. **If you're new**: install **one complete pack** (`mattpocock/skills` if you do TypeScript, otherwise `everything-claude-code` for the Swiss army knife). Work with it for 2-3 weeks before adding anything.
2. **If you have a specific pain point**: grab the matching single-purpose skill (`caveman` for an exploding bill, `claude-mem` for lost context, `graphify` to navigate a large codebase).
3. **If you want to understand the ecosystem**: spend 30 minutes on `ComposioHQ/awesome-claude-skills` and `hesreallyhim/awesome-claude-code`. You'll come out with a clear view.

<Callout type="warning" title="Don't stack 12 packs">
Multiple packs at once = convention conflicts + longer loading time + Claude doesn't know which skill to prioritize. **One pack at a time**. If you want to try a new pack, disable the previous one.
</Callout>
