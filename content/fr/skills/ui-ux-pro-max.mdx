## La promesse, citée mot pour mot

Là où [Impeccable](/skills/impeccable) apprend à Claude un vocabulaire de designer, UI UX Pro Max prend l'angle opposé : il fournit un catalogue. Au lieu de raisonner sur des principes, le skill propose des choix concrets, un style nommé, une palette assortie à un secteur, une paire de polices déjà testée. Claude pioche dans ce catalogue au lieu d'improviser.

## Ce que ça change en pratique

**Sans le skill**, demander "un dashboard analytics pour une fintech" à Claude Code donne souvent une grille de cartes grises, un bleu générique et des graphiques par défaut sans cohérence chromatique.

**Avec le skill**, le même prompt s'appuie sur une palette pensée pour la fintech (tons sobres, un accent de confiance), un style nommé (par exemple un minimalisme à forte densité d'information), une paire de polices lisible en petits corps, et un type de graphique recommandé pour la donnée affichée. Le résultat n'est pas magique, mais il part d'un socle cohérent au lieu d'une page blanche.

La différence avec Impeccable se résume ainsi : Impeccable filtre ce qui est moche, UI UX Pro Max propose ce qui marche. Les deux sont complémentaires.

```bash
uipro init --ai claude
```

## Workflow type avec Claude Code

<Steps>
<Step title="Cadrer le produit" stepNumber={1}>
Donner le contexte produit : "tableau de bord pour une app de fitness, audience grand public, ton énergique". Le skill associe ce contexte à une catégorie et à une palette.
</Step>

<Step title="Choisir un style" stepNumber={2}>
Demander à Claude de proposer 2 ou 3 styles parmi les 67, avec leur palette et leur paire de polices. Tu tranches sur une option.
</Step>

<Step title="Générer" stepNumber={3}>
Claude produit le composant ou la page dans ton stack (React, Vue, Flutter...) en appliquant le style retenu, ses tokens et ses guidelines UX.
</Step>

<Step title="Valider visuellement" stepNumber={4} isLast>
Vérifier le rendu avec [Playwright MCP](/mcp/) : screenshot mobile et desktop, contrôle des contrastes, ajustements. C'est l'étape que le skill seul ne couvre pas.
</Step>
</Steps>

## Cas d'échec à connaître

**Cas 1 — Trop de choix tue le choix.** Avec 67 styles et 161 palettes, un brief flou pousse Claude à hésiter ou à mélanger des registres. Le catalogue n'aide que si tu fixes une direction. Sans contrainte, tu obtiens un patchwork.

**Cas 2 — Design system maison déjà en place.** Si ton projet impose ses propres tokens, le catalogue UI UX Pro Max devient redondant, voire contradictoire. Dans ce cas, le skill sert au mieux à explorer des idées en amont, pas à produire le code final.

## Pour qui ?

- **Recommandé** si tu démarres une interface de zéro et que tu veux un point de départ visuel cohérent sans réfléchir toi-même à la palette et aux polices, surtout sur un stack non-React (Flutter, SwiftUI, Vue).
- **Recommandé en complément** d'[Impeccable](/skills/impeccable) : UI UX Pro Max propose la direction, Impeccable filtre les anti-patterns. Le combo couvre les deux faiblesses.
- **À éviter** si tu as déjà un design system strict : tu te battras contre le catalogue plutôt que de t'appuyer dessus.
