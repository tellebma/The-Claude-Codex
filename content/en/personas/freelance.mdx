## Votre profil

Vous travaillez seul ou en petite equipe. Votre temps est votre ressource la plus precieuse : chaque heure economisee est une heure facturable (ou une heure de repos). Vous cherchez des gains concrets de productivite, pas de la theorie.

Ce parcours va droit au but. Installation rapide, configuration optimale, puis les outils et workflows qui font vraiment gagner du temps.

<Callout type="info" title="Temps estime">
Comptez 2 heures pour les etapes essentielles (1 a 6). Les etapes 7 et 8 sont des bonus pour aller plus loin.
</Callout>

## Votre itineraire

<Steps>
<Step title="Installer en 5 minutes" stepNumber={1}>

Pas de temps a perdre : installez Claude Code et configurez votre environnement le plus vite possible.

**Lire** : [Prerequis et installation](/getting-started/installation), puis [Configuration de l'environnement](/getting-started/environment-setup)

</Step>

<Step title="Generer votre configuration sur mesure" stepNumber={2}>

Le configurateur interactif genere un CLAUDE.md, un settings.json et une configuration MCP adaptes a votre stack et vos habitudes. C'est le moyen le plus rapide de bien configurer Claude Code.

**Lire** : [Configurateur interactif](/configurator)

</Step>

<Step title="Maitriser le prompting efficace" stepNumber={3}>

Un bon prompt vous fait gagner des minutes a chaque interaction. Apprenez les bases, puis les directives qui font la difference.

**Lire** : [Les bases du prompting](/prompting/basics), puis [Directives avancees](/prompting/directives)

</Step>

<Step title="Installer les MCP de productivite" stepNumber={4}>

Les MCP connectent Claude Code a vos outils du quotidien. Gmail, Slack, GitHub, bases de donnees : choisissez ceux qui correspondent a votre workflow.

**Lire** : [Comprendre les MCP](/mcp/what-are-mcps), puis [Top MCP productivite](/mcp/best-productivity), puis [Top MCP developpement](/mcp/best-development)

</Step>

<Step title="Ajouter les plugins essentiels" stepNumber={5}>

Les plugins enrichissent Claude Code avec des fonctionnalites specialisees. Commencez par les essentiels, puis explorez ceux qui correspondent a votre metier.

**Lire** : [Comprendre les plugins](/plugins/what-are-plugins), puis [Top plugins essentiels](/plugins/best-essential)

</Step>

<Step title="Automatiser avec les Skills" stepNumber={6}>

Les Skills transforment des workflows repetitifs en commandes reutilisables. Code review, generation de tests, documentation : creez vos propres recettes.

**Lire** : [Qu'est-ce qu'un Skill ?](/skills/what-are-skills), puis [Creer un Skill custom](/skills/create-custom)

</Step>

<Step title="Optimiser vos couts" stepNumber={7}>

En tant qu'independant, chaque euro compte. Comprenez le systeme de facturation, comparez les plans et appliquez les strategies d'optimisation.

**Lire** : [Couts reels de Claude Code](/content/couts-reels-claude-code)

</Step>

<Step title="Creer des workflows MCP complets" stepNumber={8}>

Combinez plusieurs MCP dans un workflow automatise. Par exemple : recevoir un ticket GitHub, analyser le code avec Context7, implementer le fix et ouvrir une PR, le tout en une seule commande.

**Lire** : [Premier workflow MCP](/mcp/first-workflow), puis [Prompt chaining et orchestration](/prompting/chaining-and-agents)

</Step>
</Steps>
