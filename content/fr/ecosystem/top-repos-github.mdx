## Pourquoi cette page

Quand l'écosystème explose, il faut une **carte**. Les awesome lists GitHub (43k+ ★ chacune) sont déjà très bien, mais elles listent tout sans hiérarchie. Cette page filtre : on garde uniquement les repos qui ont à la fois **traction sociale** (étoiles) et **valeur d'usage réelle** (description claire, mises à jour récentes, communauté active).

Chaque entrée indique le nombre d'étoiles GitHub mesuré le jour de publication. Les chiffres bougent vite (parfois +10k★ par semaine sur les repos qui virent viraux), donc ils servent de **signal d'ordre de grandeur**, pas de vérité absolue.

<Callout type="info" title="Méthodo de curation">
Critères de sélection : 1) au moins 25k★ ou une niche unique non remplaçable, 2) commit récent (< 90 jours), 3) description claire en anglais, 4) absence de spam IA-généré, 5) usage réel documenté dans un README ou un blog. Les repos qui ont juste un README et 0 vraie traction sont écartés même si > 50k★.
</Callout>

## Comment l'utiliser

Trois usages typiques :

1. **Décider d'adopter un harness** : commencer par `everything-claude-code` (le plus complet) ou `gstack` (rôles définis, plus simple). Ne pas empiler deux harness différents.
2. **Combler un manque précis** : besoin de mémoire ? `claude-mem`. Besoin de réduire la facture ? `caveman` + `claude-code-router`. Besoin d'orchestrer plusieurs agents ? `ruflo`.
3. **Comprendre les internals** : lire `system-prompts-and-models-of-ai-tools` une fois, puis revenir sur les patterns décrits dans les awesome lists.

<Callout type="tip" title="Avant d'installer 12 outils">
La paralysie par accumulation est réelle. Choisis **1 harness** + **1 outil mémoire** + **1 awesome list comme référence**. Tu pourras toujours en ajouter plus tard. Mieux vaut maîtriser 3 outils que survoler 12.
</Callout>
