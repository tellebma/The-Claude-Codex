## D'où vient ce retour

Cet article est basé sur un mois d'utilisation de Claude Code sur un projet réel (refonte graphique du site The Claude Codex, avril-mai 2026). Les quatre skills présentés n'ont pas tous été testés avec la même intensité : Impeccable et Playwright MCP ont été utilisés quotidiennement, Taste Skill sur deux sprints, UI UX Pro Max et Huashu Design sur des sessions ponctuelles de validation.

Cette gradation est volontaire. Un comparatif honnête vaut mieux qu'un listicle qui prétend avoir tout testé en profondeur.

## Ce que Claude Code fait sans aide

Avant de parler outils, il faut planter le décor. Voici ce que produit Claude Code sur du front-end quand on ne l'outille pas :

- **Le bon** : du code propre, accessible dans les grandes lignes, qui compile et qui marche. Sur de la logique d'interface (états, formulaires, navigation), c'est solide.
- **Le moyen** : des palettes plates, le bleu Tailwind 500 par défaut, la police Inter partout, peu de hiérarchie typographique. Fonctionnel mais générique.
- **Le pas terrible** : le fameux "AI slop" visuel. Gradient violet-rose, trois cartes alignées, hero centré interchangeable d'un site à l'autre, animations absentes. Tu reconnais une page générée par IA au premier coup d'œil.

Le problème n'est pas que Claude code mal. C'est qu'un prompt vague produit un design vague. Un skill design, c'est comme donner à Claude un manuel de style avant qu'il commence à dessiner. Sans manuel, il improvise selon ses habitudes. Avec le manuel, il a des règles précises : telle hiérarchie typographique, telle palette, tel espacement. Le résultat est systématiquement plus cohérent.

## Les quatre skills, en une phrase chacun

### Impeccable

Apprend à Claude le vocabulaire d'un designer senior et applique 23 commandes plus des règles anti-patterns déterministes. C'est l'investissement le plus court pour le plus grand saut qualitatif. [Fiche détaillée Impeccable](/skills/impeccable).

### UI UX Pro Max

Fournit un catalogue : 67 styles, 161 palettes alignées par produit, 57 paires de polices, sur 15 stacks. Idéal pour partir d'une page blanche avec un socle cohérent. [Fiche détaillée UI UX Pro Max](/skills/ui-ux-pro-max).

### Taste Skill

Une famille de sous-skills par registre visuel (minimaliste, brutaliste, image-to-code). Le bon choix quand tu jongles entre des styles très différents. [Fiche détaillée Taste Skill](/skills/taste-skill).

### Huashu Design

Orienté prototypes HTML natifs et présentations. Sous licence MIT depuis le 2026-05-14 (le projet était auparavant sous licence commerciale payante, la licence a déjà changé une fois : re-vérifie avant un usage commercial). [Fiche détaillée Huashu Design](/skills/huashu-design).

## Pourquoi Playwright MCP est le pivot

Aucun de ces skills ne voit le résultat qu'il produit. Ils écrivent du code, point. C'est là qu'intervient Playwright MCP.

Playwright MCP, c'est le miroir que tu tends à Claude après qu'il a habillé un composant. Sans miroir, Claude travaille à l'aveugle. Avec, il voit exactement ce que le navigateur rend sur mobile et sur desktop, et peut corriger ce qui cloche avant que tu ouvres toi-même le navigateur.

La boucle skill design + Playwright, c'est le même processus qu'un designer qui dessine une maquette, l'imprime, la colle au mur, recule de deux mètres, voit le problème, et corrige. Sauf que la boucle prend 30 secondes au lieu de 30 minutes. Le détail du workflow, étape par étape, fera l'objet d'une page dédiée "Designer avec Claude Code + Playwright MCP".

## Ce que cette stack ne fait pas

Pour rester honnête, voici les limites réelles :

- Elle ne remplace pas un designer. Les skills élèvent le niveau de base, ils ne produisent pas une direction artistique originale à ta place.
- Elle ne garantit pas un rendu pixel-perfect. Playwright montre le rendu, mais la critique reste humaine ou semi-automatique.
- Elle ne couvre pas le design produit (parcours, wireframes UX en amont). C'est de l'habillage et du polish, pas de la recherche utilisateur.
- Les compteurs d'étoiles bougent vite et certains gonflent. Ne choisis pas sur le nombre d'étoiles, choisis sur ce que l'outil produit chez toi.
- Les skills évoluent au rythme de quelques semaines. Une commande citée aujourd'hui peut changer de nom. Vérifie toujours le repo officiel.

## Verdict prescriptif

Si tu démarres aujourd'hui sur un projet front-end avec Claude Code et que tu n'as jamais installé de skill design : installe Impeccable et active Playwright MCP. C'est le couple à plus fort rendement. Ajoute UI UX Pro Max le jour où tu démarres une interface complète de zéro, et Taste Skill le jour où tu dois jongler entre plusieurs styles. Garde Huashu Design pour les prototypes : il est passé en MIT, mais sa licence a déjà changé une fois, vérifie-la avant tout usage commercial.
