## Overview

Claude Code reads several environment variables at startup. They let you configure the tool without modifying `settings.json` files, which is ideal for CI/CD environments and automated deployments.

<Callout type="info" title="Environment variable priority">
Environment variables generally have **equal or lower** priority compared to options explicitly set in `settings.json`. Check the documentation for each option to know its exact precedence.
</Callout>

---

## Essential variables

### `ANTHROPIC_API_KEY`

| Field | Value |
|-------|-------|
| **Description** | Anthropic API key for authentication |
| **Required** | Yes (unless logged in via `claude login`) |
| **Default** | None |
| **Format** | `sk-ant-api03-...` |

```bash
# In your ~/.bashrc, ~/.zshrc or ~/.profile
export ANTHROPIC_API_KEY="sk-ant-api03-xxxxxxxxxxxx"

# For a single session
ANTHROPIC_API_KEY="sk-ant-..." claude --print "hello"

# In CI/CD (GitHub Actions)
# Set in Settings > Secrets > ANTHROPIC_API_KEY
# Then in the workflow:
env:
  ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

<Callout type="warning" title="API key security">
Never commit `ANTHROPIC_API_KEY` in a versioned file. Use a secret manager (AWS Secrets Manager, HashiCorp Vault, GitHub Secrets) in production. Rotate the key immediately if it gets exposed.
</Callout>

---

### `CLAUDE_MODEL`

| Field | Value |
|-------|-------|
| **Description** | Default Claude model to use |
| **Required** | No |
| **Default** | `claude-sonnet-4-6` |
| **Format** | Anthropic model identifier |

```bash
# Use Haiku for quick sessions (3x cheaper)
export CLAUDE_MODEL="claude-haiku-4-5"

# Use Opus for complex tasks
export CLAUDE_MODEL="claude-opus-4-6"

# Override for a single command
CLAUDE_MODEL="claude-opus-4-6" claude --print "complex task"
```

Common model identifiers:
- `claude-haiku-4-5`: Fast and affordable
- `claude-sonnet-4-6`: Recommended balance
- `claude-opus-4-6`: Deepest reasoning

---

### `MAX_THINKING_TOKENS`

| Field | Value |
|-------|-------|
| **Description** | Token budget allocated for internal reasoning (Extended Thinking) |
| **Required** | No |
| **Default** | `31999` (when Extended Thinking is enabled) |
| **Format** | Integer |

```bash
# Reduce the budget to limit costs
export MAX_THINKING_TOKENS=10000

# Maximum budget for complex tasks
export MAX_THINKING_TOKENS=31999

# Disable thinking for a session (minimum value)
export MAX_THINKING_TOKENS=1000
```

Lowering this value reduces reasoning depth and associated costs. Only relevant when `alwaysThinkingEnabled: true` or when Extended Thinking is manually enabled.

## Network and proxy variables

### `ANTHROPIC_BASE_URL`

| Field | Value |
|-------|-------|
| **Description** | Custom base URL for Anthropic API calls |
| **Required** | No |
| **Default** | `https://api.anthropic.com` |
| **Format** | Full URL |

```bash
# Enterprise proxy
export ANTHROPIC_BASE_URL="https://proxy.my-company.com/anthropic"

# Local server (development/test)
export ANTHROPIC_BASE_URL="http://localhost:8080"

# Compatible with settings.json variables
# (ANTHROPIC_BASE_URL overrides customApiUrl)
```

---

### `HTTP_PROXY` / `HTTPS_PROXY`

| Field | Value |
|-------|-------|
| **Description** | HTTP/HTTPS proxy for all outgoing connections |
| **Required** | No |
| **Default** | None |
| **Format** | `http://[user:pass@]host:port` |

```bash
# Proxy without authentication
export HTTP_PROXY="http://proxy.company.com:8080"
export HTTPS_PROXY="http://proxy.company.com:8080"

# Proxy with authentication
export HTTPS_PROXY="http://user:password@proxy.company.com:8080"

# SOCKS5 proxy
export HTTPS_PROXY="socks5://proxy.company.com:1080"
```

---

### `NO_PROXY`

| Field | Value |
|-------|-------|
| **Description** | Domains excluded from the proxy (comma-separated list) |
| **Required** | No |
| **Default** | None |
| **Format** | Comma-separated domain list |

```bash
# Exclude local network from the proxy
export NO_PROXY="localhost,127.0.0.1,*.intranet.company.com"

# Combine with HTTPS_PROXY
export HTTPS_PROXY="http://proxy.company.com:8080"
export NO_PROXY="localhost,127.0.0.1"
```

## Behavior variables

### `DISABLE_AUTOUPDATER`

| Field | Value |
|-------|-------|
| **Description** | Disable automatic update checking and installation |
| **Required** | No |
| **Default** | Not set (auto-update enabled) |
| **Format** | `1` or `true` to disable |

```bash
# Disable automatic updates
export DISABLE_AUTOUPDATER=1

# Useful in CI/CD to pin the Claude Code version
# and avoid unwanted updates in production
```

Recommended in CI/CD for build reproducibility. Use a fixed version of Claude Code in your pipelines.

---

### `CLAUDE_CODE_MAX_OUTPUT_TOKENS`

| Field | Value |
|-------|-------|
| **Description** | Maximum number of tokens in the generated response |
| **Required** | No |
| **Default** | Model limit (varies by model) |
| **Format** | Integer |

```bash
# Limit long responses
export CLAUDE_CODE_MAX_OUTPUT_TOKENS=4096

# Maximum for tasks needing long responses
export CLAUDE_CODE_MAX_OUTPUT_TOKENS=16000
```

---

### `CLAUDE_CONFIG_DIR`

| Field | Value |
|-------|-------|
| **Description** | Alternative directory for Claude Code configuration files |
| **Required** | No |
| **Default** | `~/.claude/` |
| **Format** | Absolute path to a directory |

```bash
# Use an alternative configuration directory
export CLAUDE_CONFIG_DIR="/opt/claude-config"

# Useful for multi-user environments
# or per-project isolated configurations
export CLAUDE_CONFIG_DIR="$(pwd)/.claude-config"

# In CI/CD, for a completely isolated configuration
export CLAUDE_CONFIG_DIR="/tmp/claude-ci-config"
```

## Configuration examples by context

### Local development

```bash
# ~/.zshrc or ~/.bashrc
export ANTHROPIC_API_KEY="sk-ant-api03-..."
export CLAUDE_MODEL="claude-sonnet-4-6"

# Optional: Extended Thinking with reduced budget
export MAX_THINKING_TOKENS=10000
```

### CI/CD (GitHub Actions)

```yaml
# .github/workflows/claude.yml
jobs:
  claude-task:
    runs-on: ubuntu-latest
    env:
      ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
      CLAUDE_MODEL: claude-haiku-4-5
      DISABLE_AUTOUPDATER: "1"
      CLAUDE_CODE_MAX_OUTPUT_TOKENS: "4096"
    steps:
      - uses: actions/checkout@v4
      - run: npm install -g @anthropic-ai/claude-code
      - run: claude --print --max-turns 5 "Check code quality"
```

### Enterprise environment with proxy

```bash
# /etc/environment or system profile
ANTHROPIC_API_KEY="sk-ant-..."
ANTHROPIC_BASE_URL="https://ai-proxy.my-company.com"
HTTPS_PROXY="http://web-proxy.my-company.com:8080"
NO_PROXY="localhost,127.0.0.1,*.internal.my-company.com"
DISABLE_AUTOUPDATER=1
CLAUDE_CONFIG_DIR="/opt/claude-code/config"
```

## Extended reference: less common variables

### `ANTHROPIC_AUTH_TOKEN`

Alternative authentication token used by some enterprise proxies that don't accept the standard `ANTHROPIC_API_KEY` header. When set, Claude Code sends it as `Authorization: Bearer <token>`.

```bash
export ANTHROPIC_AUTH_TOKEN="bearer-token-from-proxy"
```

### `ANTHROPIC_MODEL`

Alias accepted in addition to `CLAUDE_MODEL`. Some tutorials and third-party tools use `ANTHROPIC_MODEL`; both work and point to the same setting.

```bash
export ANTHROPIC_MODEL="claude-sonnet-4-5"
```

### `CLAUDE_CODE_USE_BEDROCK` / `CLAUDE_CODE_USE_VERTEX`

Route API calls through AWS Bedrock or Google Vertex AI instead of the Anthropic API directly. Required when your organization mandates one of these providers.

```bash
# AWS Bedrock
export CLAUDE_CODE_USE_BEDROCK=1
export AWS_REGION="us-east-1"

# Google Vertex AI
export CLAUDE_CODE_USE_VERTEX=1
export CLOUD_ML_REGION="us-central1"
export ANTHROPIC_VERTEX_PROJECT_ID="your-gcp-project"
```

### `BASH_DEFAULT_TIMEOUT_MS` / `BASH_MAX_TIMEOUT_MS`

Control how long the Bash tool waits before killing a long-running command. Useful for CI jobs that run slow builds.

```bash
export BASH_DEFAULT_TIMEOUT_MS=120000   # 2 min default
export BASH_MAX_TIMEOUT_MS=600000       # 10 min hard cap
```

### `BASH_MAX_OUTPUT_LENGTH`

Maximum output (in characters) captured from a single Bash command. Prevents huge logs from blowing up the context window.

```bash
export BASH_MAX_OUTPUT_LENGTH=30000
```

### `MCP_TIMEOUT` / `MCP_TOOL_TIMEOUT`

Timeouts (ms) for MCP server connection and individual tool calls. Raise these when connecting to slow remote MCPs.

```bash
export MCP_TIMEOUT=30000         # startup handshake
export MCP_TOOL_TIMEOUT=60000    # single tool call
```

### `MAX_MCP_OUTPUT_TOKENS`

Token budget for the combined output of all MCP tool calls in a single turn.

```bash
export MAX_MCP_OUTPUT_TOKENS=25000
```

### `CLAUDE_CODE_DISABLE_TERMINAL_TITLE`

Stops Claude Code from updating the terminal window title. Useful for tmux users and custom prompts.

```bash
export CLAUDE_CODE_DISABLE_TERMINAL_TITLE=1
```

### `DISABLE_TELEMETRY` / `DISABLE_ERROR_REPORTING`

Disable anonymous usage telemetry and Sentry error reports respectively. Both are opt-out.

```bash
export DISABLE_TELEMETRY=1
export DISABLE_ERROR_REPORTING=1
```

## Using environment variables in VS Code (`claudecode.environmentVariables`)

The official Claude Code extension for VS Code exposes a `claudecode.environmentVariables` setting that injects variables into every Claude Code process spawned from the editor. This is the cleanest way to set per-project values without polluting your global shell.

Open `settings.json` (User or Workspace) and add:

```json
{
  "claudecode.environmentVariables": {
    "ANTHROPIC_API_KEY": "sk-ant-api03-...",
    "ANTHROPIC_BASE_URL": "https://proxy.company.com/anthropic",
    "CLAUDE_MODEL": "claude-sonnet-4-5",
    "DISABLE_AUTOUPDATER": "1",
    "NO_PROXY": "localhost,127.0.0.1,*.intranet"
  }
}
```

Workspace-level `.vscode/settings.json` is ideal for committing non-secret config (model, proxy, timeouts) while keeping `ANTHROPIC_API_KEY` in User settings only.

<Callout type="warning" title="Never commit secrets">
Workspace `settings.json` is usually committed to git. Keep `ANTHROPIC_API_KEY` in User settings or use `${env:ANTHROPIC_API_KEY}` to read from the shell environment instead of hardcoding it.
</Callout>

## Precedence order (which source wins?)

When the same variable is defined in multiple places, Claude Code resolves them in this order — later wins:

1. System environment (`~/.bashrc`, `~/.zshrc`, `/etc/environment`)
2. Per-session shell export
3. `.env` file in the project directory
4. VS Code `claudecode.environmentVariables` (User then Workspace)
5. `settings.json` option with the same meaning (e.g. `customApiUrl` vs `ANTHROPIC_BASE_URL`)
6. CLI flag (`--model`, `--max-turns`, etc.)

CLI flags always win. `settings.json` values generally override environment variables when a direct equivalent exists — always check the documentation for each specific option.

## FAQ

### Where do I set `ANTHROPIC_API_KEY`?

For local dev, export it in `~/.zshrc` or `~/.bashrc`. For CI/CD (GitHub Actions, GitLab CI), store it as an encrypted secret and pass it via the `env:` block. Never commit it to git or hardcode it in `settings.json`.

### Why does my `ANTHROPIC_BASE_URL` get ignored?

Check that `HTTPS_PROXY` isn't overriding the route. If you use a corporate proxy, set both `ANTHROPIC_BASE_URL` (to the Anthropic-compatible endpoint) and `NO_PROXY` (to include that endpoint) so the proxy doesn't swallow the request.

### What's the difference between `CLAUDE_MODEL` and `ANTHROPIC_MODEL`?

They're aliases — Claude Code accepts both. Stick with `CLAUDE_MODEL` in new setups for consistency with the official docs.

### Can I disable Extended Thinking via environment variable?

Set `MAX_THINKING_TOKENS=1000` (the minimum). This effectively disables deep reasoning while keeping the feature functional. To completely disable it, remove `alwaysThinkingEnabled: true` from `settings.json`.

### How do I pin a specific Claude Code version in CI?

Set `DISABLE_AUTOUPDATER=1` and install a specific version: `npm install -g @anthropic-ai/claude-code@x.y.z`. This guarantees build reproducibility and avoids surprises from auto-updates.

### Do environment variables leak into Claude's context?

No. Environment variables configure the Claude Code client itself, not the conversation. Claude never sees `ANTHROPIC_API_KEY` or proxy settings unless you explicitly include them in a prompt.
