## TL;DR

- **MCP** (Model Context Protocol) is the canonical way to plug external tools and data into Claude Code. In 2026, the open-source standard moved from experimental to dominant.
- Aggregated references: [`punkpeye/awesome-mcp-servers`](https://github.com/punkpeye/awesome-mcp-servers) (86k★) and [`modelcontextprotocol/servers`](https://github.com/modelcontextprotocol/servers) (85k★, official repo).
- 6 categories cover 80% of needs: **dev (GitHub, Chrome DevTools, Playwright, Serena)**, **data (Supabase)**, **design (shadcn, Figma)**, **web nav (Context7)**, **automation (n8n, activepieces)**, **builder frameworks (FastMCP)**.
- Advice: start with 3 servers tied to your stack, not 30 at once. MCPs consume context and can slow Claude down.

<Callout type="info" title="What is an MCP exactly">
An MCP server exposes a set of tools (functions) and resources (data) that Claude can call. The protocol is standardized: any compliant server works with Claude Code, Cursor, Codex, Gemini CLI, etc. See [`/mcp/what-are-mcps`](/mcp/what-are-mcps) for the detail.
</Callout>

## How to choose

Three principles:

1. **Don't plug everything**: each active MCP consumes context at startup. Limit yourself to 5-7 simultaneous servers, otherwise Claude slows down.
2. **Profiles by context**: use **MCP profiles** to have a "dev" setup different from "SEO audit" or "security audit" (cf. [`/advanced/mcp-profiles`](/advanced/mcp-profiles)).
3. **Trust the official ones**: for production, prefer vendor MCPs (GitHub, Microsoft, Supabase, Anthropic). Community MCPs are great to experiment.

<Callout type="warning" title="MCP security">
An MCP can access sensitive resources (filesystem, database, browser). Before installing a non-official MCP, source code audit required. Never run an MCP requesting API tokens in local dev without checking what it does with them.
</Callout>
